# roadview-to-3d 기획서 — 로드뷰 명소 캡처 → VARCO 3D 에셋 파이프라인

> 작성: 2026-06-11 | 용도: 한강오염탐사(OilBean) "정화된 한강" 씬의 실존 명소 3D 에셋 제작

## 1. 목표

게임 기획서의 "하류에서 상류로 해금되는 구역" 구조에 맞춰, **정화 후 보여줄 아름다운 한강 명소**를
실제 로드뷰에서 캡처해 VARCO image-to-3d로 게임용 저폴리 3D 모델을 만든다.

```
[카카오맵 로드뷰] → [브라우저 자동화 캡처] → [피사체 크롭] → [(선택) VARCO Upscale]
→ [VARCO image-to-3d (저폴리)] → [GLB 다운로드] → [Unity 임포트 (glTFast)]
```

## 2. 가능성 검토 결과 (2026-06-11 실증)

| 항목 | 결과 | 비고 |
|---|---|---|
| 로드뷰 좌표 딥링크(`/link/roadview/lat,lng`) | ❌ 불가 | 데스크톱에선 좌표 무시됨 |
| 장소명 검색 → 로드뷰 진입 | ✅ 가능 | 두 가지 경로 검증 (아래 3-A/3-B) |
| UI 없는 클린 캡처 (1920×1080) | ✅ 가능 | 오버레이 전부 DOM 숨김 처리 |
| 장소 핀 없는 캡처 | ✅ 가능 | 커버리지 클릭 진입 시 핀 없음 |
| 헤딩 회전 | ✅ 가능 | 키보드 홀드(≈7°/s), 시각 확인 필수 |
| 파노라마 이동 | ✅ 가능 | 바닥 화살표 클릭 |
| 로드뷰 줌(FOV) | ❌ 무효 | `.btn_zoomin` 효과 없음 → 크롭으로 대체 |
| 피사체 크롭 품질 | ✅ 충분 | 중앙 배치 시 선명 (가장자리는 블러) |
| VARCO Upscale 연계 | ✅ 가능 | 크롭 해상도 2~6배 보강 |
| VARCO image-to-3d 연계 | ✅ 가능 | 비동기(requestId 폴링), GLB 출력 |
| Unity GLB 임포트 | ✅ 가능 | OilBean에 gltfast 6.19.0 설치됨 |

**대안 검토**: 네이버 거리뷰(미검증, 카카오 충분), Kakao JS SDK(API키 필요, UI 자동화로 충분), Google Street View(한국 커버리지 빈약).

## 3. 검증된 핵심 레시피

상세 스니펫은 `references/browser-recipes.md` 참조. 요약:

- **3-A. 핀 있는 빠른 진입**: 검색 → `PlaceItem a.roadview` 클릭 → 파노라마 직행 (장소 핀이 캔버스에 그려짐 — 피사체와 겹치면 부적합)
- **3-B. 핀 없는 진입(권장)**: 검색 → 첫 결과 `link_name` 클릭(지도 센터링) → `a.rv` 토글(커버리지 표시) → `scripts/find_line.py`로 얇은 파란선 좌표 탐지 → 클릭 → 파노라마
- **클린 캡처**: 파노라마 상태에서만 position:absolute/fixed 오버레이 일괄 숨김 (지도 상태에서 실행하면 타일이 사라짐!)
- **회전**: `canvas.focus()`(JS) 후 ArrowLeft/Right 홀드. **클릭으로 포커스 잡으면 파노라마가 이동해버림** — 절대 금지
- **로딩 감지**: 중앙 200×200 클립 스크린샷의 PNG 크기 > 8KB 될 때까지 폴링 (검은 화면은 PNG가 작음)

## 4. 폴리곤 예산 (경량화 원칙)

카메라 컬링을 나중에 넣더라도 기본은 가볍게. `target_face_num`으로 직접 제어:

| 단계 | target_face_num | 용도 |
|---|---|---|
| 1차 (기본) | **10,000** | 배경 명소 — 원거리 실루엣 위주 |
| 2차 | 50,000 | 1차 품질 미달 시 |
| 3차 | 150,000 | 주요 랜드마크 클로즈업 한정 |
| 금지선 | 300,000(API 최대) | 사용하지 않음 |

판정 기준: 실루엣 무너짐/지붕 구멍/텍스처 뭉개짐이 보이면 한 단계 상향. 같은 seed로 면수만 바꿔 비교.

## 6. 리스크 / 주의

- **저작권**: 카카오 로드뷰 이미지는 Kakao 자산. 프로토타입·내부 레퍼런스 용도로 사용하고, 상용 출시 시 라이선스 검토 필요. 생성된 3D는 변형 저작물이므로 동일 주의.
- **UI 변경 취약성**: 카카오맵 셀렉터(`a.rv`, `.PlaceItem` 등)가 개편되면 레시피 갱신 필요. 스크린샷 기반 시각 확인 루프가 안전망.
- **크레딧 비용**: image-to-3d 호출당 크레딧 소모. 같은 피사체 반복 생성 최소화(seed 고정, 면수만 변경).
- **원거리 피사체**: 로드뷰가 물가 건물에 접근 불가한 경우(섬 위 건물 등) 크롭 해상도가 낮아짐 → Upscale 2x로 보강.
